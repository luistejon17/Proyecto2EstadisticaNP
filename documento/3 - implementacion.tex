\subsection{Estadístico \texorpdfstring{$T_n$}{T\_n} (Schuster--Barker)}

La diferencia $F_n(x)-sF_n(x;\theta)$ es una función escalera que solo cambia en los puntos del conjunto $\{X_i\}\cup\{2\theta-X_i\}$, por lo que el supremo se alcanza necesariamente en alguno de esos $2n$ puntos. Usando la identidad
\[
F_n(x)-sF_n(x;\theta) \;=\; \tfrac{1}{2}\bigl(F_n(x) - 1 + F_n((2\theta-x)^-)\bigr),
\]
la implementación vectoriza el cálculo sobre $K$ valores de $\theta$ simultáneamente con dos llamadas a \texttt{searchsorted} sobre la muestra ordenada:

\begin{lstlisting}[language=Python]
def Tn_multi(sample, thetas):
    x_sorted = np.sort(sample)
    n = x_sorted.size
    refl = 2.0 * thetas[:, None] - x_sorted[None, :]
    grid = np.concatenate([np.broadcast_to(x_sorted, refl.shape),
                           refl], axis=1)              # (K, 2n)

    Fn       = np.searchsorted(x_sorted, grid, side='right') / n
    arg      = 2.0 * thetas[:, None] - grid
    Fn_minus = np.searchsorted(x_sorted, arg,  side='left')  / n

    diff = 0.5 * (Fn - 1.0 + Fn_minus)
    return np.sqrt(n) * np.max(np.abs(diff), axis=1)
\end{lstlisting}

Esto evita los $\mathcal{O}(K\cdot 4n^2)$ del enfoque ingenuo y reduce el cómputo a $\mathcal{O}(K\cdot n\log n)$ amortizado, lo que es crítico para el estimador $\hat\theta_{\min}$ porque debe evaluar $T_n$ sobre toda una rejilla.

\subsection{Estadístico \texorpdfstring{$S_n$}{S\_n} (función característica empírica)}

La integral de penalización \eqref{eq:Sn} se aproxima sobre un grid uniforme $\{t_1,\dots,t_K\}$ con $K=301$. La clave para que el barrido sobre $\theta$ sea barato es la siguiente: dado que $c_n(t)=a_n(t)+i\,b_n(t)$ no depende de $\theta$, se tiene
\[
\bigl|c_n(t)e^{-it\theta}-c_s(t)\bigr|^2 \;=\; 2|c_n(t)|\bigl(|c_n(t)|-A_n(t;\theta)\bigr),\qquad A_n(t;\theta)=a_n(t)\cos(t\theta)+b_n(t)\sin(t\theta).
\]
Por lo tanto $a_n,b_n,|c_n|$ se precalculan una vez por muestra, y luego se barren los $M$ valores de $\theta$ con costo $\mathcal{O}(MK)$ en vez de $\mathcal{O}(MKn)$:

\begin{lstlisting}[language=Python]
def Sn_multi_theta(sample, thetas, w_fn, q=2, t_grid=None):
    if t_grid is None: t_grid = make_t_grid(w_fn)
    arg_raw = t_grid[:, None] * sample[None, :]          # (K, n)
    a_n = np.mean(np.cos(arg_raw), axis=1)               # (K,)
    b_n = np.mean(np.sin(arg_raw), axis=1)               # (K,)
    abs_cn = np.sqrt(a_n**2 + b_n**2)                    # (K,)

    tth = t_grid[:, None] * thetas[None, :]              # (K, M)
    A = a_n[:, None]*np.cos(tth) + b_n[:, None]*np.sin(tth)
    diff_sq = 2.0 * abs_cn[:, None] * (abs_cn[:, None] - A)
    np.maximum(diff_sq, 0.0, out=diff_sq)                # estabilidad

    integrand = diff_sq if q == 2 else np.sqrt(diff_sq)
    return np.trapezoid(integrand * w_fn.fn(t_grid)[:, None],
                        t_grid, axis=0)                   # (M,)
\end{lstlisting}

En perfiles preliminares esta vectorización redujo el tiempo de $\hat\theta_{\min,S_n}$ en aproximadamente un orden de magnitud frente a una implementación basada en bucles puros.

\subsection{Estimadores del centro de simetría}

La mediana y la media afeitada se implementan directamente con \texttt{numpy.median} y \texttt{scipy.stats.trim\_mean} (con \texttt{proportiontocut=0.1}). El estimador $\hat\theta_{\min}$ requiere optimización numérica y se implementa de forma distinta para cada estadístico, aprovechando su estructura matemática (Sección~\ref{sec:estim-theta}).

\subsubsection{argmin de \texorpdfstring{$T_n$}{Tn}: algoritmo de Schuster-Narvarte}\label{impl:argmin-Tn}

Como $T_n$ es escalonada con saltos exactamente en $\mathcal{W}_n=\{(X_i+X_j)/2:i\le j\}$ (Proposición~\ref{prop:Tn-steps}), el mínimo global se alcanza en algún $W_{ij}$. En lugar de evaluar los $\mathcal{O}(n^2)$ promedios de Walsh, se usa el \emph{algoritmo combinatorio exacto de Schuster-Narvarte}: para la muestra ordenada $x_0\le\dots\le x_{n-1}$, se itera $k=0,1,\dots$ calculando
\[
  m(k)=\max_{i+j=(n-1)-k} \tfrac{x_i+x_j}{2}, \qquad
  M(k)=\min_{i+j=(n-1)+k} \tfrac{x_i+x_j}{2}.
\]
Al primer $L$ con $m(L)\le M(L)$ se devuelve $\hat\theta=(m(L)+M(L))/2$. La complejidad es $\mathcal{O}(L\cdot n)$; empíricamente $L$ mediana es $3$--$8$ para $n\in\{20,\dots,160\}$, por lo que el algoritmo corre en tiempo $\mathcal{O}(n\log n)$ en la práctica.
\newpage
\begin{lstlisting}[language=Python]
def theta_argmin_schuster_narvarte(sample):
    x = np.sort(sample)
    n = x.size
    for k in range(n):
        # Anti-diagonal inferior: i+j = (n-1)-k
        s_lo = (n-1) - k
        i_lo = np.arange(max(0, s_lo-(n-1)), s_lo//2 + 1)
        m_k = 0.5 * float(np.max(x[i_lo] + x[s_lo - i_lo])) if i_lo.size else -np.inf

        # Anti-diagonal superior: i+j = (n-1)+k
        s_hi = (n-1) + k
        i_hi = np.arange(k, s_hi//2 + 1)
        M_k = 0.5 * float(np.min(x[i_hi] + x[s_hi - i_hi])) if i_hi.size else np.inf

        if m_k <= M_k:
            return 0.5 * (m_k + M_k)
    return float(np.median(x))
\end{lstlisting}
La misma función se usa tanto en la muestra original como en cada remuestra bootstrap: las remuestras provienen de $sF_n(\hat\theta)$, que es simétrica por construcción, por lo que $L$ es pequeño también en el bootstrap.

Otro enfoque que intentamos fue hallar los $K$ promedios de Walsh más cercanos a la mediana o a un estimador a la mediana, y asumir que el argmin estaría en esta ventana, así evitando calcular los $O(n^2)$ promedios de Walsh. Este enfoque aunque funcionó para $K=8n$,  de media resultó de 4 a 400 veces más lento que usar el algoritmo de Schuster-Narvarte explicado en \cite{schuster1987bootstrap}.

\subsubsection{argmin de \texorpdfstring{$S_n$}{Sn}: L-BFGS-B con gradiente analítico}\label{impl:argmin-Sn}

Para $q=2$, la Proposición~\ref{prop:leibniz-Sn} garantiza que $S_n$ es $C^1$ y proporciona la fórmula cerrada del gradiente. Esto habilita un esquema de quasi-Newton (Sección~\ref{sec:estim-theta}) que evita el barrido sobre toda una grilla. La rutina precomputa $(a_n,b_n,|c_n|)$ una sola vez ($\mathcal{O}(Kn)$) y cada iteración de \texttt{L-BFGS-B} cuesta $\mathcal{O}(K)$ — una llamada al par valor–gradiente:

\begin{lstlisting}[language=Python]
def _Sn_value_and_grad_q2(theta, a_n, b_n, abs_cn, w_vals, t_grid):
    # A_n y B_n son la parte real e imaginaria de c_n(t) * exp(-i t theta)
    A = a_n * np.cos(t_grid * theta) + b_n * np.sin(t_grid * theta)
    B = -a_n * np.sin(t_grid * theta) + b_n * np.cos(t_grid * theta)
    # Valor: S_n = integral de 2 |c_n| (|c_n| - A) w dt
    integrand = 2.0 * abs_cn * (abs_cn - A)
    np.maximum(integrand, 0.0, out=integrand)        # estabilidad
    S_n = np.trapezoid(integrand * w_vals, t_grid)
    # Gradiente: dS/dtheta = -2 integral t |c_n| B(t,theta) w dt
    dS_n = np.trapezoid(-2.0 * t_grid * abs_cn * B * w_vals, t_grid)
    return float(S_n), float(dS_n)

def theta_argmin_Sn(sample, w_fn, q=2, t_grid=None):
    if q != 2:
        return _theta_argmin_Sn_grid(sample, w_fn, q, ...)   # fallback
    a_n, b_n, abs_cn, w_vals = _Sn_precompute(sample, w_fn, t_grid)
    res = minimize(
        lambda th: _Sn_value_and_grad_q2(th[0], a_n, b_n, abs_cn,
                                          w_vals, t_grid),
        x0=np.array([np.median(sample)]),
        jac=True, method='L-BFGS-B',
        bounds=[(sample.min()-pad, sample.max()+pad)],
        options={'gtol': 1e-7, 'ftol': 1e-10, 'maxiter': 50},
    )
    return float(res.x[0])
\end{lstlisting}

El punto de partida es la mediana muestral (Corolario~\ref{cor:H0-gradient-zero} sugiere que el verdadero centro es un punto estacionario asintótico, y la mediana es $\sqrt n$-consistente). Para $q=1$ el integrando contiene un $\sqrt{\cdot}$ no diferenciable en sus ceros, por lo que se recurre a la búsqueda grid + Brent (\texttt{theta\_argmin\_Sn\_grid}).


\subsubsection{argmin de \texorpdfstring{$S_n$}{Sn} para \texorpdfstring{$q=1$}{q=1}: selección discreta con Pyomo}\label{impl:argmin-Sn-pyomo}

Para $q=1$, el estadístico $S_n$ sigue siendo evaluable numéricamente, pero el criterio no es diferenciable en los puntos donde \(c_n(t)e^{-it\theta}-|c_n(t)|=0\).
Por esta razón, no se usa el gradiente analítico de la Proposición~\ref{prop:leibniz-Sn}, que fue derivado específicamente para el caso $q=2$. En su lugar, se construye una grilla finita de candidatos para $\theta$, se evalúa $S_n(\theta)$ en cada candidato mediante la rutina vectorizada \texttt{Sn\_multi\_theta}, y luego se selecciona el candidato con menor valor objetivo.

Esta selección discreta se implementa opcionalmente con \texttt{Pyomo}. Si los candidatos son $\theta_1,\dots,\theta_m$ y los valores correspondientes del estadístico son $v_i=S_n(\theta_i)$, el problema se formula como el siguiente programa lineal entero mixto:
\[
\min_{y_1,\dots,y_m} \sum_{i=1}^m v_i y_i,
\qquad
\sum_{i=1}^m y_i=1,
\qquad
y_i\in\{0,1\}.
\]
La variable binaria $y_i$ indica si el candidato $\theta_i$ es seleccionado. Como exactamente una variable debe tomar el valor $1$, la solución óptima escoge el candidato con menor valor de $S_n$.

En la implementación, \texttt{Pyomo} no se importa al cargar el paquete, sino únicamente cuando se invoca explícitamente el estimador \texttt{argmin\_pyomo}. El solver se selecciona automáticamente entre los disponibles en la lista
\[
\texttt{appsi\_highs},\quad \texttt{highs},\quad \texttt{glpk},\quad \texttt{cbc}.
\]
Si ninguno está instalado, la rutina informa el error y sugiere instalar un solver compatible, por ejemplo \texttt{highspy}. Esta estrategia permite incluir el caso $(q=1,\hat\theta_{\min})$ dentro del estudio, evitando el uso de derivadas y manteniendo una formulación reproducible del problema de selección del argmin.


\subsection{Procedimiento bootstrap}

Ambos tests comparten el mismo esqueleto bootstrap: (i)~calcular el estadístico observado con el estimador sobre la muestra original, (ii)~construir el soporte simetrizado $\{X_i,\,2\hat\theta-X_i\}_{i=1}^{n}$, (iii)~remuestrear $B$ veces y recalcular estimador y estadístico, y (iv)~obtener el p-valor con la corrección de continuidad estándar: $\hat p=(1+\#\{b:T_n^{*(b)}\ge T_n^{\text{obs}}\})/(B+1)$.

Para \texttt{"argmin"}, tanto la muestra original como cada remuestra usan \texttt{theta\_argmin\_schuster\_narvarte}: como las remuestras son simétricas (vienen de $sF_n(\hat\theta)$), el algoritmo converge en pocos pasos sin necesidad de anclas externas ni ventanas heurísticas.

\begin{lstlisting}[language=Python]
def bootstrap_test_Tn(sample, estimator='argmin', B=199, rng=None):
    rng = rng or np.random.default_rng()
    est_fn = ESTIMATORS[estimator]   # p.ej. theta_argmin_schuster_narvarte
    theta_hat = est_fn(sample)
    T_obs = Tn_statistic(sample, theta_hat)
    support = np.concatenate([sample, 2 * theta_hat - sample])

    T_boot = np.empty(B)
    for b in range(B):
        y = rng.choice(support, size=sample.size, replace=True)
        theta_b = est_fn(y)          # exacto: SN converge rapido en y simetrica
        T_boot[b] = Tn_statistic(y, theta_b)

    p_val = (1.0 + np.sum(T_boot >= T_obs)) / (B + 1.0)
    return BootstrapResult(T_obs, p_val, theta_hat, T_boot, estimator, B)
\end{lstlisting}

La función \texttt{bootstrap\_test\_Sn} sigue un patrón similar pero con una optimización adicional: para los estimadores mediana y media afeitada, cuyo cálculo se vectoriza trivialmente sobre el eje de réplicas, se genera la matriz de remuestras $(B,n)$ de una vez y se evalúa $S_n$ en lote mediante \texttt{\_sn\_boot\_batch}. Esta función itera sobre mini-lotes de tamaño $B_\text{sub}=\lfloor200\,\text{MB}/(K\cdot n\cdot 8)\rfloor$ para no exceder $\sim\!200$~MB de RAM:

\begin{lstlisting}[language=Python]
def _sn_boot_batch(Y, thetas, w_fn, q, t_grid):
    """Evalua S_n para B remuestras (B,n) con thetas (B,)."""
    B_total, n = Y.shape; K = t_grid.size
    w_vals = w_fn.fn(t_grid)              # (K,)
    B_sub = max(1, int(200_000_000 // (K * n * 8)))
    S_boot = np.empty(B_total)
    for start in range(0, B_total, B_sub):
        end = min(start + B_sub, B_total)
        Yb = Y[start:end]; tb = thetas[start:end]
        arg = t_grid[None,:,None] * Yb[:,None,:]  # (bs,K,n)
        a_n = np.mean(np.cos(arg), axis=2)        # (bs,K)
        b_n = np.mean(np.sin(arg), axis=2)
        abs_cn = np.sqrt(a_n**2 + b_n**2)
        tth = t_grid[None,:] * tb[:,None]          # (bs,K)
        A = a_n*np.cos(tth) + b_n*np.sin(tth)
        diff_sq = 2.0*abs_cn*(abs_cn - A)
        np.maximum(diff_sq, 0.0, out=diff_sq)
        integrand = diff_sq if q==2 else np.sqrt(diff_sq)
        S_boot[start:end] = np.trapezoid(
            integrand * w_vals[None,:], t_grid, axis=1)
    return S_boot
\end{lstlisting}

Para el estimador argmin, la optimización iterativa (\texttt{L-BFGS-B}) impide vectorizar: se mantiene el bucle secuencial, con el punto de partida $x_0=\mathrm{mediana}(Y^{*(b)})$ recalculado en cada remuestra (la mediana de la remuestra es $\sqrt{n}$-consistente para el centro de $sF_n(\hat\theta)$).

\subsection{Paralelización del estudio Monte Carlo}

Cada celda del diseño experimental (distribución $\times$ $n$ $\times$ estimador $\times$ $(q,w)$ para $S_n$) requiere $R$ réplicas independientes; en total $18\,000$ tareas para $T_n$ y $60\,000$ para $S_n$ (la combinación $q{=}1$ con argmin se excluye, reduciendo de $72\,000$ a $60\,000$). Como cada réplica es independiente, se paralelizan con \texttt{ProcessPoolExecutor} usando $\#\text{cores}-1$ workers. Para garantizar reproducibilidad y evitar correlación entre semillas, a cada tarea $k$ se le asigna la semilla $\texttt{seed} + k$:

\begin{lstlisting}[language=Python]
def _single_replica(spec_idx, n, estimator, B, base_seed, task_idx):
    spec = _SPEC_REGISTRY[spec_idx]
    rng = np.random.default_rng(base_seed + task_idx)
    sample = spec.sampler(n, rng)
    t0 = time.perf_counter()
    res = bootstrap_test_Tn(sample, estimator=estimator, B=B, rng=rng)
    return {'dist': spec.name, 'n': n, 'estimator': estimator,
            'statistic': res.statistic, 'p_value': res.p_value,
            'time_s': time.perf_counter() - t0,
            'under_h0': spec.under_h0}
\end{lstlisting}
 Con 11 cores activos, el estudio completo de $T_n$ ($18\,000$ tests, $B=199$) corre en $\approx 1.5$ minutos (gracias al algoritmo exacto de Schuster-Narvarte para el argmin) y el de $S_n$ ($60\,000$ tests, $K=301$, $B=199$) en $\approx 20$--$30$ minutos. La reducción de $B=500$ a $B=199$ y la vectorización del bucle bootstrap para los estimadores mediana y media afeitada (eliminando el bucle de Python y evaluando $S_n$ en lote sobre matrices $(B,K,n)$) son las principales ganancias de velocidad.

\subsection{Verificación empírica con pruebas unitarias}

Antes de lanzar el estudio masivo se desarrollaron pruebas unitarias para verificar la corrección de cada componente. Los tests más importantes son:
\begin{itemize}
  \item \textbf{Vectorización contra referencia ingenua}: se compara \texttt{Tn\_multi} y \texttt{Sn\_multi\_theta} contra una implementación naive en bucles puros, en muestras pequeñas; deben coincidir hasta tolerancia $10^{-10}$.
  \item \textbf{Invariancia por traslación}: $T_n$ y $S_n$ evaluados en $\theta=c$ sobre la muestra $X$ deben coincidir con los evaluados en $\theta=0$ sobre $X-c$.
  \item \textbf{Pesos integran a 1}: se verifica $\int w(t)\,dt = 1$ por cuadratura, dentro de tolerancia.
  \item \textbf{$\hat\theta_{\min}$ recupera el centro}: en muestras grandes simétricas (Normal, Uniforme) el argmin de $T_n$ y $S_n$ debe estar a menos de $0.1$ del verdadero centro.
  \item \textbf{Sanidad del bootstrap}: bajo $H_0$ los p-valores empíricos son aproximadamente uniformes en $(0,1)$ (la figura del p-valor en la sección de resultados sirve como verificación visual); bajo $H_a$ con $n$ grande la potencia tiende a $1$.
\end{itemize}
Las pruebas se encuentran en \texttt{tests/} y se ejecutan con \texttt{python tests/test\_statistics\_ks.py} y \texttt{python tests/test\_characteristic\_fn.py}. Las~$22$ pruebas pasan satisfactoriamente con la implementación actual.
